About the Dataset

This is a simulated credit card transaction dataset containing legitimate and fraud transactions from the duration January 1, 2019 â€“ June, 2020. It covers credit cards of 1000 customers doing transactions with a pool of 800 merchants.

Data Processing

The csv file, consisting of 23 columns and 1296675 rows, was loaded into Jupyter Notebook. The dataset was narrowed down by removing irrelevant columns.


Balancing the data

The dataset is imbalanced with 1289169 legitimate transactions and only 7506 fraud transactions. It has a large volume of genuine transactions and comparatively very few fraudulent transactions. The dataset was balanced using the upsampling technique.


Visualize the data

As the next step we wanted to visualize the distribution of the transactions. Histogram was plotted to visualize the distribution of fraud transactions based on amount.

Feature Engineering

The customer date of birth was replaced by the person's current age. The categorical features, category and job were label encoded and converted to numeric. The dataset was splitted using SKLearn into training and testing datasets. The splitted data was scaled using Standard Scalar.


Feature Selection

We used Random Forest Classifier and XGBoost Classifier for identifying the weight of each feature on the fraud transactions and prioritise feature importance. Both the algorithms classified amount as the most important feature and category as the second most important feature in determining fraud transactions.


Feature Correlation

Next step is to identify if there is any correlation between the feaures that detemine the classification of the transactions. So we plotted feature correaltion diagram. The feature correlation diagram represents the relation between independent features.


Model Selection

We used various algortihms to train the dataset. The models used were Logistic Regression Model, Gaussian Naive Bayes Model, Gradient Boosting Regressor Model, K-Nearest Neighbors (KNN) Model and Support Vector Machines (SVM) Model. In order to improve performance and optimize the models, the models were tuned using hyper-parameter tuning with grid search. The performance of the models were evaluated by charting confusion matrix and class matrix. Compared to other models, Logistic Regression model predicted the fraud transactions with a highest accuracy of 89%.



Save the Model

We selected Logistic Regression Model as the final model to make predictions. The trained model was saved as .sav file which was loaded to the flask app to be used for predicting the fraud transactions in the test dataset.
